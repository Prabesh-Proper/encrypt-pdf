⚙ How It Works

  • Password is converted into a secure encryption key using PBKDF2.
  • File is padded using PKCS7 to fit AES block size.
  • Encryption uses AES-256 in CBC mode.
  • Encrypted file contains Salt + IV + Ciphertext.
  • During decryption, salt and IV are extracted automatically.

🔐 Best Security Practices

  • Use a strong password (minimum 8–12 characters).
  • Avoid common or reused passwords.
  • Use HTTPS if deploying publicly.
  • Disable debug mode in production.
